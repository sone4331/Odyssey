# Odyssey 技术学习指南

这套文档服务于已具备 Unity 基础、希望用 Odyssey 准备游戏客户端岗位的读者。它不替代源码：每一页都先给出系统要解决的问题，再给出实际入口和调用链。阅读时请同时打开对应 C# 文件的中文 XML 注释。

## 推荐顺序

1. [01 - 项目与运行](01-项目与运行.md)：先把项目正确跑起来，知道可演示什么。
2. [02 - 架构与装配](02-架构与装配.md)：建立程序集、生命周期和依赖方向的全局地图。
3. [03 - 玩家与战斗](03-玩家与战斗.md)：追踪输入如何变成移动、技能、伤害和 UI。
4. [04 - AI 与遭遇战](04-AI与遭遇战.md)：理解敌人如何感知、决策和开启关卡门禁。
5. [05 - 配置、存档与编辑器](05-配置存档与编辑器.md)：理解数据如何进入运行时，以及如何安全保存。
6. [06 - 双人合作联机](06-双人合作联机.md)：区分 Owner 表现、Host 权威和网络同步。
7. [07 - 测试、调试与扩展](07-测试调试与扩展.md)：把理解转化为可验证的工程改动。
8. [08 - 面试准备](08-面试准备.md)：用项目事实组织面试回答，而不是背术语。

## 阅读原则

- 先沿一条用户可见的链路读完，再回头看抽象层。推荐从“按下攻击键”开始。
- `Core → Gameplay → Unity → Network/Editor` 是依赖与职责方向；上层可以适配下层，领域规则不能反向依赖 Unity。
- 代码注释解释边界、顺序和取舍；方法名和赋值等直接语义不重复解释。
- 本文档描述当前作品集范围，不把它误读为完整线上游戏框架。Relay、匹配、确定性同步和通用热更新均不在范围内。
